Next, I needed to create service/factory in order to pull that information. I name my application 'app' (because I'm clever like that) and create a service called 'projectsService.' Our array with the JSON data is called 'projects.' We create a function called 'getProjects' that can be used by our controllers later.

var app = angular.module('projectsApp', ['ngRoute']);

app.service('projectsService', function () {
    this.getProjects = function () {
        return projects;
    };...

Creating a controller: The C in MVC

Next, we need to create a controller to pull this information. We name it 'ProjectsController' and specify the $scope and our service in the parameters. I create an initialization function (init()) out of habit and so I can control the initialization of my controller. It's certainly not necessary for such a simple example, but it is good to control how everything is initialized.

app.controller('ProjectsController', function ($scope, projectsService) {
    init();

    function init() {
        	$scope.projects = projectsService.getProjects();
    }
});

Pages for each project: Loading another view

The goal was to display all of the projects in a grid and when you click on one, you can see additional information on that specific project. In order to that, we need another partial HTML file that knows which project info to load. This is why the links above pass in the project's id. We can then use that to find the project we want. The first step is to rework both our controller and service to find the right project. We add a new function to our service called 

this.getSpecific = function (id) {
	//search through every item in our projects object
    for(i=0;i<projects.length;i++)
    {
    	//does our current project match our parameter
        if(projects[i].id == id)
            return projects[i];
    }
};

Then, we need to change our controller to accept parameters. Then, if we have a parameter, it needs to call the appropriate function from our service.

app.controller('ProjectsController', function ($scope, $routeParams, projectsService) {
    init();

    function init() {
		//if there are paramaters (id) call getSpecific
		var hasParams = Boolean(Object.keys($routeParams).length);
		if(hasParams)
		{
			$scope.projects = projectsService.getSpecific($routeParams.id);
		}
		//otherwise, pull them all
		else
		{
			$scope.projects = projectsService.getProjects();
		}
    }
});

Controlling our routes: Controlling when these pages are shown

With AngularJS, we need to specify when views are shown by managing our routes. In my app.js file, I next specify when each page is loaded. Here, we can really control how everything is loaded and this makes it really easy to change the site quickly if needed. We create our default (projects.html) and make it redirect there if nothing is specified. We create one for our details page and then our final one is for our code examples page (which is what you are viewing now). 

app.config(['$routeProvider',
  function($routeProvider) {
    $routeProvider.
      when('/projects', {
        templateUrl: 'partials/projects.html',
        controller: 'ProjectsController'
      }).
      when('/info/:id', {
        templateUrl: 'partials/details.html',
        controller: 'ProjectsController'
      }).
	  when('/code/:id', {
        templateUrl: 'code/examples.html',
        controller: 'CodePageController'
      }).
      otherwise({
        redirectTo: '/projects'
      });
  }]);

My code pages: Using what I had already created

In my old portfolio, I had written up a lot of the examples you see here. I really didn't want to recreate everything from scratch. Luckily, Angular allows me to 'include' html files in a view. I simply stripped out everything but the body of the page, and included that in my new partial: examples.html.

<main ng-controller="CodePageController">
    <div ng-include="activePage.url"></div>
</main>

As you can see, I need a new controller to control how this information is pulled. The reason for this is that I need to control which HTML doc is loaded based on the project ID at that time. For that reason, I need a new array that controlls all of the URLs I need. We can use the same functions from our service to pull the project, but now we want to specify what the 'activePage' is based on the 'exampleInt' I specified in our projects object. Our 'allPages' array corresponds those values to the appropriate HTML. It works kind of like a view, although there is certainly more overhead here. However, it saved me from having to recreate the wheel with this project.

app.controller('CodePageController', function ($scope, $routeParams, projectsService) {
    init();

    function init() {
		//if there are paramaters (id) call getSpecific
		//there should always be params on this page, but it is best to check
		var hasParams = Boolean(Object.keys($routeParams).length);
		if(hasParams)
		{
			$scope.projects = projectsService.getSpecific($routeParams.id);
		}
		//otherwise, pull them all
		else
		{
        	$scope.projects = projectsService.getProjects();
		}
		
		//our array which will hold all of our URL values
		$scope.allPages = [
		{
			url: 'code/adminjs.html' //0
		},
		{
			url: 'code/draft.html' //1
		},
		{
			url: 'code/tileCode.html' //2
		},
		{
			url: 'code/submitCode.html' //3
		},
		{
			url: 'code/storedProc.html' //4
		},
		{
			url: 'code/hallofheroes.html' //5
		},
		{
			url: 'code/scrapStreet.html' //6
		},
		{
			url: 'code/angular.html' //7
		},
		{
			url: 'code/angular.html' //7
		}]
		
		//our JSON list of projects will give us the int we need in order to load the right page
		$scope.activePage = $scope.allPages[$scope.projects.exampleInt];
    }
});
